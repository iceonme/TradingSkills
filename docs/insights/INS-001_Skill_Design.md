# INS-001: Trading Skills 设计思想与分类规范

> **状态**: 🟢 ACTIVE
> **日期**: 2026-02-26
> **所属模块**: TradingSkills (TS) 专栏

## 1. Trading Skills 设计理念

Trading Skills (TS) 是为加密交易场景专门设计的标准化能力工具箱。它严格遵循开放的大语言模型技能协议（如 Anthropic Skills / MCP）。
在 TS 的设计哲学中，技能不再是硬编码在单体程序内部的函数，而是独立的、可即插即用的资源包。每个 Skill 包含一个向 AI 解释使用说明的 `SKILL.md`（系统提示词），以及用于底层运转的执行脚本。通过此设计，多智能体系统获得了类似“生态配件”的扩展能力。

## 2. Skill 的双轨分类模型

为了实现极致的架构解耦与防冗余（DRY），TS 内部将技能从逻辑上严格划分为两类：**交易执行型 (Execution Skills)** 与 **策略型 (Strategy Skills)**。

### 2.1 基础设施型 Skill (Infrastructure Skills)
此类 Skill 是系统的底层基建，分为“读”和“写”两个标准化维度：
- **交易执行型 (`trading_exe`)**: 负责“写”操作。类似于“交易 ERC 标准”，提供统一的挂单、撤单、资产查询接口。
- **数据枢纽型 (`data_hub`)**: 负责“读”操作。提供统一的数据拉取标准，涵盖 K 线行情、实时 Tick、舆情热度及宏观日历等数据源。它让上层策略无需关心数据是从 OKX、Binance 还是 Twitter 爬虫来的。

### 2.2 认知与情报型 Skill (Intelligence Skills)
负责对原始数据进行初加工，为 Agent 提供更高维度的判断素材。
- **舆情/宏观工具**: 专门负责从非结构化数据中提取特征。
- **数据分析包**: 提供统计学、机器学习等分析工具集。

### 2.3 策略型 Skill (Strategy Skills)
此类 Skill 是纯粹的交易逻辑资产，负责基于输入数据生成买卖信号或触发高频机器人。
- **工作流**: 调用 `data_hub` 获取输入数据 -> 进行逻辑计算 -> 调用 `trading_exe` 完成动作。

## 3. 结论

将策略智慧从底层执行器中剥离，把 `trading_exe` 确立为交易生态的公共基础协议，是构建加密智能体生态的关键步骤。这种声明式（Agent）与执行式（Scripts）相分离的做法，最大化了 TradingSkills 在开源世界的重用性与可维护性。
