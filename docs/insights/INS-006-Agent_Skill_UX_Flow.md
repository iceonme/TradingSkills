# INS-006: Agentic UX Flow - 智能体如何发现与拼装交易技能

**日期**: 2026-03-01
**主题**: 探讨在全新的 Skill 生态下，AI Agent 及其人类主人是如何循序渐进地获得、发现并串联量化交易能力的。
**状态**: [已确认] 加入架构指导思想。

## 1. 核心问题
当我们将庞大的单体量化脚本爆改成“数据卡带 + 策略卡带 + 执行卡带 + 游戏机(Runner)”后，**站在 AI Agent 的用户体验视角**，它是如何理解并使用这些碎片化的工具的？

## 2. 智能体交易能力的三阶赋能路径 (Agent UX Flow)

根据我们的探讨，Agent 获得完整量化闭环能力的过程，类似于一个“乐高积木发现与拼装”的游戏。

### 阶段一：获得“手足” (执行层赋能)
- **场景**: 用户在 ClawHub 或应用商店配置 Agent，勾选了“赋予交易执行能力”。
- **动作**: Agent 被安装了基础的 `okx_trading_exe`（执行网关模块）。
- **能力表现**: 此时的 Agent 是一个被动的**发单客服**。它自己没有想法，只会听从人类的自然语言指令（如：“帮我市价买入 100 U的比特币”），然后调取底层 API 执行。

### 阶段二：发现“大脑” (策略的自我进修)
- **场景**: `okx_trading_exe` 作为一个智能卡带，在它的 `SKILL.md`（给大模型看的说明书）中埋入了引导语：“*如果你需要脱离人类，进行 7x24x365 的全自动量化盯盘，你需要去市场安装 `Runner` 和相应的 `Strategy Skill`*”。
- **动作**: Agent 读取说明书后，向人类主人**主动提议**：“为了更好地为您赚钱，我建议去 ClawHub 下载并安装『网格策略卡带』与『行情获取卡带』”。

### 阶段三：终极形态 (全自动数字生命)
- **场景**: 人类同意并为 Agent 补齐了所有的组件库（数据、策略、执行，以及作为串联底座的 Runner 脚本）。
- **动作**: Agent 此时不再一笔笔地手动操作交易接口，而是转变为一个“量化机房管理员”。它执行启动命令 `python scripts/live_runner_demo.py`，让系统在后台自行拉数据、算 RSI、判断网格界限并异步发单。
- **能力表现**: Agent 退居幕后。它平时只需要观察 Runner 输出的日志，并在必要时打断程序，更换另一张策略卡带（如行情从震荡变为单边，抽出网格卡带，插入均线趋势卡带），或者微调 Runner 里面的时间周期参数。

## 3. 架构解耦的根本意义
这条 UX 逻辑线完美诠释了为什么要极度解耦：
**不同的 Agent 在生命周期的不同阶段，对能力颗粒度的需求是完全不一样的。**
统一的 Interface Schema (`core/types.py`) 保证了，只要插口一致，Agent 就可以从最简陋的手工下单工具人，一路升级为挂载了复杂回测/实盘双域编排器的高级量化资本家。
