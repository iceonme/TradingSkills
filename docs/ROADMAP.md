## v0.1 挑战赛版本（当前进行中）

### 目标
构建交易挑战赛平台，让不同类型"交易者"在**相同历史数据**上竞技，验证 MAS 架构有效性，发现最优策略组合。

> 💡 **为什么是挑战赛？**  
> 这是通往最终愿景（个人 AI 交易助手）的第一步。通过挑战赛我们可以：
> 1. **验证 MAS 协作效率**：测试"信息层 Feed 协作"是否比"LLM 直接工具调用"更高效、更准。
> 2. 发现什么样的策略组合最有效
> 3. 积累历史回测数据，训练更好的 Agent
> 4. 为后续产品提供可信的"策略库"

### 核心需求

#### 1. 参赛者系统
**需求**: 支持多种类型的交易参与者同台竞技

| 参赛者 | 优先级 | 说明 |
|--------|--------|------|
| **DCA 定投选手** | P0 | ✅ 已实现 |
| **Grid 高抛低吸** | P0 | ✅ 已实现并深度优化（动态跟随、递归分仓、1m 短线支持）|
| **MAS 小队** | P0 | ✅ 基础框架已建立 |
| **LLM 单兵** | P1 | ✅ 已实现，待进一步增强 |
| **自定义 MAS** | P2 | 📋 待开发 |

**验收标准**:
- [x] 定义统一的 `Contestant` 接口
- [x] 参赛者可以独立初始化、接收行情、返回交易决策
- [x] 参赛者之间互不干扰，公平竞技

#### 2. 数据回放引擎
**需求**: 基于历史数据回放，模拟真实市场环境

**已有基础**: `lib/core/replay-engine.ts`（基础实现）

**需完善**:
- [x] 支持 2025 全年 BTC 1分钟数据回放
- [x] 动态频率图表同步渲染
- [x] 完善 VirtualClock 注入机制

#### 3. MAS 小队运转
**需求**: 完整的盯盘 → 研判 → 交易流程

**盯盘模块** (技术分析员):
- [x] 计算基础指标（RSI, MA, MACD）- 通过 analysis-tools.ts 实现
- [x] 识别交易信号（超买超卖、金叉死叉、背离）- LLM Indicator/Strategy 已支持
- [ ] 生成预测（"预计 24h 内回调 3-5%"）
- [ ] 基于记忆避免重复发信号

**研判模块** (PA):
- [ ] 收集技术分析员的信号
- [ ] 实现加权决策算法
- [ ] 生成交易指令（买/卖/持仓、数量、理由）
- [ ] 记录决策理由

**交易模块**:
- [ ] 模拟执行交易
- [ ] 更新持仓和资金
- [ ] 记录交易历史

#### 4. 评估系统
**需求**: 公平评估各参赛者表现

| 指标 | 说明 |
|------|------|
| 总收益率 | (期末资产 - 初始资金) / 初始资金 |
| 年化收益率 | 根据比赛时长年化 |
| 夏普比率 | 风险调整收益 |
| 最大回撤 | 从高点到低点的最大跌幅 |
| 胜率 | 盈利交易次数 / 总交易次数 |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 交易频率 | 交易次数 / 比赛天数 |

**验收标准**:
- [ ] 实时计算各参赛者资产曲线
- [ ] 生成对比报告
- [ ] 支持多维度排名

#### 5. 可视化界面
**需求**: 直观展示比赛过程和结果

**验收标准**:
- [ ] 资产曲线对比图（多参赛者叠加）
- [ ] 实时排行榜
- [ ] 交易记录时间线
- [ ] 关键指标仪表盘

### 技术需求

- **数据**: DuckDB 行情数据（已有 ✅）
- **回放**: ReplayEngine（已有 ✅，需完善）
- **框架**: TradeMind Framework（已有 ✅）

## ⭐ v0.1.x 工具化 & 进化（下阶段规划）

> **现状评估**：当前系统是**实验级别**——Grid 和 DCA 是独立的参赛选手，彼此平行，而不是 PA 可以统一调度的工具。PA 缺乏统一决策层和工具调用能力，离真正的 MVP 还有距离。本阶段目标是推进至 **MVP 水平**。

---

### 工作项一：PA 工具化完善（通往 MVP） 🔧

**背景**：Grid（高抛低吸）和 DCA（定投）本质上是两种可复用的"交易工具"。PA（Portfolio Agent）应该成为调用这些工具的决策中枢，而不是一个被动的竞技参赛者。

**目标**：

- [ ] 将 `GridContestant` 和 `DCAContestant` 的核心逻辑抽象为 **可调用的 Tool Function**
- [ ] PA 具备基本的**仓位感知**（当前持仓、盈亏状态、风险敞口）
- [ ] PA 能够基于市场状态（趋势 vs 震荡）**动态选择并调用工具**
- [ ] PA 有明确的资金管理规则（每个工具分配多少比例的资金）
- [ ] 完善 PA 的决策日志，使决策过程可解释

**PA 工具清单（目标）**：

| 工具 | 说明 | 状态 |
|------|------|------|
| `DCA(params)` | 定投工具，定时买入 | 待封装 |
| `Grid(params)` | 高抛低吸工具，网格策略 | 待封装 |
| `GetPosition()` | 查询当前持仓和盈亏 | 待实现 |
| `GetMarketState()` | 获取当前市场状态（趋势/震荡/波动率）| 待实现 |
| `SetRiskLimit(percent)` | 设置最大单笔风险 | 待实现 |

**验收标准**：
- [ ] 在 Arena 中，"PA 智能模式"能够在不同行情阶段自动选择 DCA 或 Grid 工具
- [ ] PA 的综合回测表现优于单一固定策略基准
- [ ] 决策链路（为什么选这个工具、参数是什么）在日志中清晰可见

### 工作项二：策略快速测试 & AI 监督进化 🧬

> 🔬 **这是一个研究方向，不是近期工程任务。记录在此以备将来实施。**

**愿景**：让系统能够自动对策略参数进行搜索和优化，类似深度学习的训练过程——但优化的对象是"工具/算法/模型本身"，而不是神经网络权重。

**核心比对**：

| 概念 | 深度学习 | 策略进化 |
|------|----------|----------|
| 模型 | 神经网络权重 | 策略参数（`gridLevels`, `pivotN`, `buySpacing` 等）|
| 损失函数 | Cross Entropy | Sharpe 比率、最大回撤、胜率的综合评分 |
| 优化器 | Adam / SGD | 随机搜索 → 遗传算法 → 贝叶斯优化 |
| 验证集 | Holdout Test Set | 样本外时间段（Out-of-Sample）|
| 过拟合 | 泛化能力差 | 历史好但实盘失效 |

**技术路径**：

```
训练循环
  ├── 从数据库一次性加载 N 个月 K 线（只读一次，复用）
  ├── 参数搜索器（遗传算法）生成候选参数集
  ├── 并行运行 M 个纯内存回测（无 UI 渲染）
  ├── 计算每次的综合评分
  ├── 保存结果到"进化日志表"
  ├── 输出最优参数（Top-K）
  └── 样本外时间段验证 → 防止过拟合
```

**关键工程问题**：
- **快速回测**：去除 UI 渲染、K 线数据只加载一次、Worker Threads 并行化（目标：单轮搜索 < 10 分钟）
- **评分函数**：`score = Sharpe × (1 - maxDrawdown) × winRate`（综合多维指标）
- **数据保存**：参数组合、各维度评分、分市场状态子评分、进化历史曲线

**里程碑（待规划）**:
- [ ] 设计评分函数与基准
- [ ] 实现快速回测框架（纯内存计算，无 UI）
- [ ] 接入随机搜索（第一版，简单实用）
- [ ] 实现遗传算法优化器
- [ ] 可视化进化过程（评分曲线、参数分布图）
